### Visualization and Analysis - Visualization - Making a picture or movie from data - Plots, interactivity, 3D, etc. - Intuition and exploration - Analysis - Fitting or making a model from data - Simulation, statistics, regression (machine learning), etc. - Theory and mathematics
### My Work - Department of Energy (DOE), Los Alamos National Laboratory (LANL) - http://lanl.gov - Petabytes of data from simulations and sensors - I help scientists visualize and analyze their data - My research communities - *IEEE Visualization*: http://ieeevis.org - *SIGGRAPH*: http://siggraph.org - *DOE Office of Science*: http://science.energy.gov
### Tools - I'll cover the tools that I typically use - Python, R, ParaView, and Cinema - Python and R are common in "data science" - ParaView is unique to scientific visualization - ParaView was initially created at LANL - Maintained by Kitware, Inc. - Cinema is an ongoing research project at LANL
### Similarities between Data Science and Scientific Visualization - We have big data sets - We like to use Python and R for analysis - There's a lot of hype in machine learning - We use GPUs - The scientific visualization and graphics communities did most of the initial research - We use BLAS, LAPACK, FFTW, GSL, etc. - Created by scientific and numerical methods communities -- inside Python and R libraries
### Differences -- in scientific computing and at the DOE we: - Typically favor speed over portability - *Don't* use the cloud - *Don't* use Java, we use Fortran, C++, and MPI - Supercomputers have fast, specialized networks, for large, parallel computations and leverage MPI (Message Passing Interface) - *Don't* use Hadoop, Spark, the Apache stack, etc. - *Don't* use Docker that much (yet!)
### Some other open source tools that I won't cover that I have used at some point in my career OpenGL, Spark, Flink, Dask, ggobi, Torch, gnuplot, D3, Vega/Vega-lite, Pure Data + GEM, Orange, MeshLab, VisIt, Mayavi, Processing, vvvv, xmdvtool, Plotly, charts.js, Octave, Blender, Slicer

### Scientific Visualization - IEEE VIS - https://ieeevis.org - V - VAST - I - InfoVis - S - *SciVis* (Scientific Visualization) - Usually, "3D" (physical data) meshes or volumes - At LANL, simulations, like climate modeling - In medicine, CT, MRI, etc. - Time varying, large-scale data - Dark matter cosmology, petabytes of data
### ParaView https://www.paraview.org/download/ - Back in the late 1990s/early 2000s - ParaView started at DOE LANL - VisIt started at DOE LLNL - Features - 3D, polygonal and volume rendering - Scalable, distributed parallel (supercomputers) - VTK (Visualization ToolKit) - Kitware, Inc.
### ParaView Demo - Menu `File->Open` - Many file formats - Open `Examples/can.ex2` - Select all variables - Rotate and Zoom the model - Switch between representations - Points - Wireframe - Feature Edges
### ParaView Demo - Cut (clip) - Slice - Threshold - Glyph - Pipeline view - Animate - Tools Menu - Histogram
### ParaView Demo - Calculator - Calculate squared distance - Color By - Edit Menu - Find Data: Points, `Result <= 200` - View Menu - Python Shell - `GetSources` - Start and Stop Trace
### VTK and ParaView Python - `vtkpython` and `pvpython` - Everything you can do in the GUI you can script - Stand-alone visualization programs - Programmable Filter - my most used filter - Python scripts executed in a VTK pipeline - Custom data filtering - Can directly process the data with numpy - Immediate feedback/rapid prototyping
### Open Source Data - SciVis holds scientific visualization contest - Curates data that can be used for research, etc. - Every year since 2004 (minus 2007 and 2009) - http://sciviscontest.ieeevis.org - Some LANL data is available - Simulation of an asteroid hitting the ocean - The data used for winning entry in Supercomputing contest
### DOE-oriented Scientific Visualization Research - Focus on parallelism and large-scale data - Portable Parallel Algorithms - Run on GPU or multi-core CPU transparently - In situ Algorithms - as it is running - In transit Algorithms - as the data are moving - Data compression and data reduction - Lossy compression algorithms - Data products (plots, images, etc.)
### Cinema - https://cinemascience.github.io/examples.html - https://cinemascience.github.io/publications.html - Save every plot/image that was ever needed - Assumes that visualizations are data reduction - Many scientists keep spreadsheets of their work - Viewers that can interactively show - The scientist's spreadsheet of their work as CSV - Database of every plot referenced in a CSV
